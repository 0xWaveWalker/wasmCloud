Claims(structure)

Represents a set of [RFC 7519](https://tools.ietf.org/html/rfc7519) compliant JSON Web Token claims.
expires: org.wasmcloud.model#U64 All timestamps in JWTs are stored in _seconds since the epoch_ format as described as `NumericDate` in the RFC. Corresponds to the `exp` field in a JWT. Optional.
id: String Corresponds to the `jti` field in a JWT.
issuedAt: org.wasmcloud.model#U64 The `iat` field, stored in _seconds since the epoch_
issuer: String Issuer of the token, by convention usually the public key of the _account_ that signed the token
metadata: Blob Optional custom jwt claims in the `wascap` namespace
notBefore: org.wasmcloud.model#U64 The `nbf` JWT field, indicates the time when the token becomes valid. If `None` token is valid immediately
subject: String Subject of the token, usually the public key of the _module_ corresponding to the WebAssembly file being signed

InvocationArgs(structure)

Performs an invocation that can either target an actor or a capability provider. It is up to the caller to ensure that all of the various arguments make sense for this invocation, otherwise it will fail.
actorKey: String actorKey - The public key of the actor performing the invocation. If the invocation is coming from a provider, this argument is ignored.
binding: String binding - The link name for the invocation. This will be ignored in actor-to-actor calls.
namespace: String namespace - The namespace of the operation. For provider calls, this will be something like `wasmcloud:messaging`. For actor targets, this is the public key of the target actor
payload: Blob payload - The raw bytes of the payload to be placed _inside_ the invocation envelope. Do **not** serialize an invocation for this parameter, one is created for you.
prefix: String prefix - The lattice subject prefix of the lattice into which this invocation is being sent.
providerKey: String provider_key - The public key of the capability provider involved in this invocation. This value will be ignored for actor targets.
seed: String seed - The seed signing key of the host, used for invocation anti-forgery tokens.

InvocationResponse(structure)

response: Blob response - The response, if successful

PutActorReference(operation)

Publishes a reference from an alias (OCI or call alias) to a given WasmCloudEntity. This reference will be added to the distributed cache and made available to all hosts for quick lookup. References will be checked before invocations and other administrative operations wherever possible, falling back on public keys when no reference is found.

PutLinkDefinition(operation)

Publishes a link definition to the lattice for both caching and processing. Any capability provider with the matching identity triple (key, contract, and link name) will process the link definition idempotently. If the definition exists, nothing new will happen. This function does not return a success indicator for the link processing, only an indicator for whether the link definition publication was successful.
input: PutLinkArgs
output: InvocationResponse
